📱 Caméras à distance

Tu peux utiliser n'importe quel téléphone ou autre appareil comme caméra supplémentaire :

  1. Sur le téléphone, ouvre l'URL ou flashe le QR code affiché à gauche
  2. Le téléphone demande l'autorisation caméra/micro → accepte
  3. Choisis la caméra (avant / arrière) puis clique Se connecter
  4. Le téléphone apparaît automatiquement comme nouvelle source dans le Studio

La connexion se fait en WebRTC pair-à-pair via le service PeerJS Cloud (gratuit). Une fois établie, la vidéo passe en direct entre les deux appareils.

Astuces

  • Pour le moindre délai vidéo, mets téléphone et Mac sur le même WiFi
  • Le téléphone doit garder son écran allumé pendant la diffusion (réglages → ne pas laisser l'écran s'éteindre)
  • Tu peux connecter plusieurs téléphones simultanément

❔ Aide — Studio VersetLive

Qu'est-ce que le studio ?

Le studio est une régie complète dans ton navigateur : il mixe plusieurs caméras (USB et téléphones), incruste le verset en direct, enregistre localement et diffuse en RTMP vers YouTube/Facebook/Twitch via le relais local.

En 5 minutes

  1. Ajoute une caméra — clique « + Ajouter caméra » en haut à gauche, choisis ta webcam.
  2. Ajoute un téléphone (optionnel) — clique l'icône QR code, flashe avec le téléphone, accepte l'autorisation caméra.
  3. Choisis une scène — colonne centrale, clique sur la scène (caméra seule, caméra + verset, verset plein écran, etc.). Tu peux aussi appuyer sur les touches 1–9.
  4. Affiche un verset — depuis l'onglet principal (panneau retour), sélectionne un verset → il apparaît automatiquement dans le studio.
  5. Lance le stream ou enregistre — boutons ⏺ Enregistrer ou « Démarrer » dans la section Streaming.

Conseils

  • Active le mode 🔴 Direct (Preview/Program) pour préparer la scène suivante avant de l'envoyer en direct.
  • La sortie projecteur (📽) ouvre une fenêtre à glisser sur ton second écran ou à diriger vers un câble HDMI.
  • Pour une TV sans câble, utilise 📺 TV — voir l'onglet TV.

Utiliser un téléphone comme caméra

Trois options, du plus simple au plus avancé. Tu peux mélanger : 1 webcam USB + 1 Iriun + 1 QR en même temps, c'est OK.

1. Iriun Webcam — recommandé pour un usage régulier

App tierce qui transforme ton téléphone en webcam système. Une fois installée, le téléphone apparaît dans la liste des caméras USB comme n'importe quelle webcam — aucune config dans VersetLive.

  1. Sur le Mac : télécharge l'app desktop depuis iriun.com et installe-la.
  2. Sur le téléphone : installe Iriun Webcam depuis l'App Store (iOS) ou Play Store (Android).
  3. Lance les deux apps avec Mac + téléphone sur le même Wi-Fi. La détection est automatique.
  4. Dans le Studio : clique + Ajouter caméra → la caméra Iriun apparaît dans la liste, choisis-la.
  • Pour qui : usage récurrent, mêmes téléphones tous les dimanches, qualité fiable.
  • Coût : gratuit jusqu'à 1080p avec un petit watermark Iriun. Version Pro (~15 $) pour 4K et logo retiré.
  • USB possible : au lieu du Wi-Fi, branche le téléphone en USB pour une latence quasi nulle (Android : activer le débogage USB ; iPhone : voir option 3).

2. QR Code intégré VersetLive

La solution sans rien à installer sur le téléphone.

  1. Dans le Studio, regarde la section 📱 Caméras à distance : un QR code et un code de salle s'affichent.
  2. Sur le téléphone : flashe le QR (ou tape l'URL manuellement).
  3. Accepte les autorisations caméra + micro dans le navigateur.
  4. La caméra du téléphone apparaît côté Studio dans la liste des sources.
  • Pour qui : invités, dépannage, sessions ponctuelles, téléphones qu'on n'utilise qu'une fois.
  • Coût : gratuit, zéro install.
  • Limites : à refaire à chaque session, dépend de la stabilité WebRTC du navigateur mobile.

3. Continuité de la caméra (iPhone + macOS 13+)

Apple a un système natif où l'iPhone branché en USB (ou même proche en sans-fil) devient une webcam macOS sans aucune app.

  1. iPhone connecté avec le même Apple ID que le Mac.
  2. Branche l'iPhone en USB (ou laisse-le à proximité, écran éteint).
  3. Dans le Studio : clique + Ajouter caméra → ton iPhone apparaît comme caméra système.
  • Pour qui : qualité max, latence minimale, déjà dans l'écosystème Apple.
  • Limites : iPhone + macOS 13 (Ventura) ou plus uniquement.

Conseils Wi-Fi

  • Mac et téléphones sur la même bande 5 GHz : la latence chute par rapport au 2,4 GHz.
  • Évite les VPN sur le téléphone — ils cassent la découverte locale (Iriun et QR code WebRTC).
  • Si plusieurs caméras saturent le Wi-Fi : passe-en au moins une en USB.

Diffuser vers YouTube / Facebook / Twitch

Le streaming RTMP passe par un relais Node.js local qui tourne sur ta machine (pas sur Vercel). Il diffuse simultanément vers toutes les destinations configurées.

Première fois

  1. Lance start-versetlive.command (double-clic dans le Finder) — il démarre le relais et ouvre le studio.
  2. Le navigateur peut afficher un avertissement « certificat non sécurisé » sur https://localhost:8766. C'est normal (cert auto-signé). Accepte une fois.
  3. Dans la section Streaming du studio, ajoute une destination : nom, URL RTMP (ex. rtmp://a.rtmp.youtube.com/live2), clé de stream.
  4. Clique « Démarrer » → le stream passe en direct sur toutes les destinations cochées.

Où trouver l'URL et la clé RTMP

  • YouTube : Studio YouTube → Direct → Stream → « URL du serveur de diffusion » et « Clé de diffusion ».
  • Facebook : Live Producer → Configuration de la diffusion → « URL du serveur » et « Clé de diffusion ».
  • Twitch : Tableau de bord créateur → Paramètres → Diffusion → « Clé principale de diffusion ».

Conseils

  • Les clés sont stockées en localStorage sur ta machine, jamais envoyées sur internet (sauf au relais local).
  • Tu peux désactiver/réactiver chaque destination à la volée via la case à cocher.
  • Le débit dépend du nombre de destinations × résolution. Sur fibre, 3 destinations en 1080p passent largement.

Affichage sans-fil sur une TV

Permet d'afficher le verset (ou le mix vidéo complet) sur n'importe quel écran avec un navigateur — Smart TV, Mac, iPad, console, Apple TV en mode web…

  1. Sur la TV, ouvre le navigateur et va sur versetlive.vercel.app/tv.
  2. La TV affiche un code à 4 caractères (ex. K3M9).
  3. Dans le studio, clique 📺 TV et saisis ce code.
  4. Choisis le mode : verset seul (ultra-léger, data only) ou mix vidéo complet (WebRTC).

Astuces

  • L'écran de la TV reste allumé automatiquement (Wake Lock API — Smart TV récentes).
  • Touche R sur la télécommande de la TV pour régénérer le code.
  • Tu peux connecter plusieurs écrans en parallèle (chacun avec son propre code).
  • Si le mode vidéo affiche une image figée, recharge la page TV — le navigateur peut bloquer l'autoplay au premier chargement.

Écrans de bienvenue et de remerciement

Configure une intro (avant le service) et une outro (après) avec animation et musique de fond. Une fois configurées, elles apparaissent comme scènes à part entière dans la liste.

Deux modes par écran

  • Texte animé — titre + sous-titre + logo + countdown jusqu'à l'heure de début du service. Couleurs personnalisables.
  • Vidéo importée — fichier MP4 ou WebM qui tourne en boucle.

Audio de fond

Tu peux ajouter un fichier MP3 qui se joue automatiquement quand la scène intro/outro est active. L'audio est mixé dans le flux RTMP — il sera entendu par les spectateurs.

Où sont stockés les fichiers ?

Tous les assets (logo, vidéo, MP3) sont stockés en IndexedDB sur ta machine. Rien n'est uploadé. Si tu changes d'ordinateur, il faudra ré-importer.

Setup Google Cloud (une seule fois)

Le planificateur appelle l'API YouTube depuis ton navigateur. Pour ça, Google demande un Client ID OAuth à toi (pas le mien) — ça prend 5 minutes.

  1. Va sur console.cloud.google.com et connecte-toi avec le compte Google de la chaîne YouTube de ton église.
  2. Crée un projet — barre du haut → « Nouveau projet ». Nomme-le par exemple VersetLive.
  3. Active YouTube Data API v3 — menu ☰ → « API et services » → « Bibliothèque » → cherche YouTube Data API v3 → clique Activer.
  4. Configure l'écran de consentement OAuth — « API et services » → « Écran de consentement OAuth » → Externe → remplis nom de l'app, email, etc. → ajoute le scope https://www.googleapis.com/auth/youtube. Ajoute ton compte Google dans « Utilisateurs de test ».
  5. Crée le Client ID OAuth — « API et services » → « Identifiants » → Créer des identifiants → ID client OAuth. Type : Application Web.
    • Origines JavaScript autorisées : https://versetlive.vercel.app et http://localhost:8765
    • URI de redirection : laisse vide (on utilise le flow implicit)
  6. Copie le Client ID qui s'affiche (forme 123…-abcdef.apps.googleusercontent.com) et colle-le dans le studio (bouton 📅 Planifier).

Workflow planification

  1. Clique 📅 Planifier dans la topbar du studio.
  2. Saisis ton Client ID (1ère fois seulement), clique Enregistrer puis Se connecter à Google. Une popup Google s'ouvre, accepte les autorisations.
  3. Remplis titre, description, date/heure de début, visibilité. Clique Créer le live YouTube.
  4. La destination RTMP est automatiquement ajoutée dans le studio (mais désactivée par défaut).
  5. Le jour J, ouvre le studio → clique 📅 Planifier → ton live apparaît avec un countdown. À H–0, le bouton ▶ Démarrer maintenant clignote.
  6. Clique Démarrer : active la destination, lance le stream RTMP, passe le broadcast YouTube en direct.
  7. À la fin : clique ■ Terminer le live dans la liste.

Limites

  • YouTube limite le quota d'API à 10 000 unités/jour par projet (largement suffisant pour un usage normal d'église).
  • Tant que ton appli est en « mode test » dans Google Cloud, le token expire après 7 jours et tu dois te reconnecter. Pour éviter ça, soumets l'app pour vérification (long) ou utilise toujours le même compte test.
  • La clé RTMP YouTube générée par broadcast est non réutilisable — chaque live a sa propre clé.

« La caméra du téléphone ne se connecte pas »

  • Vérifie que téléphone et Mac sont sur le même WiFi.
  • Sur le téléphone, accepte bien l'autorisation caméra/micro (icône cadenas dans Safari/Chrome).
  • Recharge la page caméra du téléphone et reflashe le QR code.
  • Vérifie que ton firewall n'empêche pas la connexion WebRTC.

« Le stream ne démarre pas »

  • Le relais local tourne-t-il ? Vérifie l'indicateur « Relais : connecté » en bas de la section Streaming.
  • Si non : relance start-versetlive.command ou relay/start.command.
  • Accepte le certificat auto-signé : visite https://localhost:8766 une fois et clique « Continuer ».
  • Vérifie la clé RTMP — un caractère manquant et la plateforme refuse silencieusement.

« Le son est en double / en écho »

  • L'audio des téléphones est désactivé par défaut (anti-feedback). Active-le source par source via le bouton 🔇/🔊 si tu en as besoin.
  • Si ton mic est aussi capté par un téléphone, coupe l'un des deux.

« L'image gèle sur la TV en mode vidéo »

  • Le navigateur de la TV bloque parfois l'autoplay vidéo avec son.
  • Touche le bouton « ▶ Toucher pour démarrer » s'il apparaît sur la TV.
  • Ouvre la console JS de la TV pour voir les erreurs (la plupart des Smart TV ne le permettent pas, mais sur Mac/iPad oui).

« Vercel ou GitHub semble en panne »

  • L'app fonctionne en local sans internet (sauf streaming/TV). Lance python3 -m http.server 8765 dans le dossier et ouvre http://localhost:8765.
